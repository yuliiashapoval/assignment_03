Щоб об'єднати інформацію з таблиць order_items та products, база даних обрала метод Hash Join. Замість складного пошуку, Postgres просто переглянув усі рядки по черзі (зробив Seq Scan) в обох таблицях. Це сталося тому, що в наших таблицях поки що дуже мало тестових даних, і системі набагато швидше перебрати їх усі напряму, ніж використовувати індекси. Спочатку база швидко знайшла потрібні позиції для замовлення, а потім так само просто підтягнула їхні назви з таблиці товарів
